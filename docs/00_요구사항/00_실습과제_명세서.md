# 실습 과제 명세서

> 실무 프로젝트처럼, **고객 요청사항 한 건**을 받아 서비스로 완성한다.

---

## 1. 과제

- 대상 서비스: 일자리 맵(JOBABA MAP) — 지도 기반 채용공고 탐색 웹 서비스
- 시작점: **`고객 요청사항.xlsx`** (발주처 요청 기능 11건)
- 목표: 요청사항을 근거로 **분석 → 설계 → 개발 → QA → 서버 배포**까지 AI와 함께 완주
- 참고 완성본: https://jobaba-map.tanauxd.com

---

## 2. 제공물

| 항목 | 내용 |
|---|---|
| `고객 요청사항.xlsx` | 발주처 요청 기능 목록(사용자/관리자/공통) |
| `data/` | 채용공고(고용24·잡코리아), 공통코드, 직종분류, 기업주소·좌표 등 CSV |
| `data/데이터_안내.md` | 데이터 설명·좌표 처리·개인정보 처리 |

> 소스코드·설계 문서·가이드는 제공하지 않는다. 요청사항과 데이터만으로 직접 끌고 간다.

---

## 3. 완주 범위

- 포함: 요구사항 분석, 화면·데이터·API 설계, 퍼블리싱, 카카오맵 연동, QA, **서버 배포**.
- 추가 구현: 관리자 통계와 채용공고 CRUD도 구현 범위에 포함했다.

---

## 4. 완료 확인

- 배포 URL(HTTPS)에서 지도·목록·검색이 동작한다.
- 프로젝트 폴더에 단계별 산출물(분석·설계·소스·테스트·배포)이 존재한다.
- (제출은 받지 않으며, 프로젝트 구조로 확인한다.)

---

## 5. 제약·가정

- 지도 라이브러리: **카카오맵**(JS 키는 본인 발급).
- 좌표: 주소 기반 **지오코딩**으로 파생(상세: `data/데이터_안내.md`).
- 데이터: 제공 CSV 사용, 실 DB 미연동. 산출물에 실제 개인정보 포함 금지.

---

## 6. 핵심 원칙

- 방향(무엇을·왜)은 본인이 쥐고, 실행(어떻게)은 AI에게 지시한다.
- 막히면 직접 붙잡지 말고 AI에게 물어 다음 단계로 넘어간다.
