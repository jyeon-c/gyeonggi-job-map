# 시스템 구성 및 데이터 흐름

## 구성

```text
브라우저/팝업 위젯
  ├─ 카카오맵 SDK
  └─ HTTPS /api
       ↓
Cloudflare → nginx(정적 파일·리버스 프록시)
                  ↓
             Spring Boot API
                  ↓
             PostgreSQL
```

## 채용 데이터 흐름

```text
제공 CSV·공통코드·직종분류
  → build-jobs-data.ps1
  → 활성/경기 필터
  → 공식 코드 매핑
  → 기업주소·기업좌표 사업자번호 조인
  → 중복·결측·형식 검증
  → jobs.json + data-quality-report.json
  → 컨테이너 기동 시 DB 교체 적재
  → /api/jobs
```

## 지도·목록 상태 흐름

- 필터/검색 변경: 서버 전체 페이지 조회 → ID 중복 방어 → 지도 범위 필터 → 목록/마커 렌더
- 지도 이동: 현재 화면 좌표를 다시 계산해 목록과 지역 집계 갱신
- 지역 배지 선택: 해당 행정구역 공고만 목록 표시, 줌 변경 시 선택 해제
- 개별 핀 선택: 해당 공고 한 건만 목록 표시, 줌 변경 시 현재 지도 영역 목록 복원
- PC 최초 위치: 사용자 동의 → Cloudflare IP 좌표 → 미제공 시 브라우저 위치
- 모바일 최초 위치: 사용자 동의 → 브라우저 GPS

## 보안 경계

- 카카오 JavaScript 키는 도메인 제한하고 런타임 설정 파일로 주입한다.
- DB/관리자 비밀번호와 SSH 키는 저장소에 커밋하지 않는다.
- API는 DTO만 반환하며 원본 개인정보 컬럼을 적재하지 않는다.
- 관리자 API만 HTTP Basic 인증, 일반 조회 API는 공개한다.
- 화면에 삽입되는 API/검색 문자열은 HTML escape 처리한다.
