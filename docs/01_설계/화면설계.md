# 화면 설계

본 프로젝트의 화면은 고객 요청사항과 구현 화면을 기준으로 HTML/CSS/JavaScript에서 직접 설계·구현했다.

## 사용자 화면

### 기본 레이아웃

- 상단: 서비스 제목, 검색창, 필터, 정책정보 메뉴
- 좌측: 채용공고 목록, 총 노출 건수, 마감 기준일
- 우측: 카카오 지도
- 모바일: 목록/지도 전환형 반응형 UI

### 지도 UI

- 넓은 줌: 지역별 `공고 수 + 지역명` 배지 표시
- 가까운 줌: 개별 공고 마커 표시
- 지역 배지 hover: 해당 시·군·구 행정경계 반투명 표시
- 지역 배지 클릭: 좌측 목록을 해당 지역 공고로 제한
- 줌 인/아웃: 지역 클릭 상태 해제 후 현재 지도 영역 목록 복원
- 공고 마커 클릭: 해당 공고 1건만 좌측 목록에 표시

### 목록 UI

- 카드 정보: 회사명, 제목, 지역, 직종, 임금, 마감, 출처, 원문 링크
- 마감 공고는 사용자 목록에서 제외
- 목록 hover 시 지도에 해당 위치 범위 표시
- `exact`/`company_address` 좌표는 작은 반경, `region_approx`는 위치 근사로 표시

### 위치·검색

- 현재 위치 버튼 제공
- 위치 동의 안내 표시
- 브라우저 위치 정확도가 낮으면 안내 토스트 표시
- 주소/장소 검색은 지도 이동 및 주변순 목록에 반영
- 직무 키워드는 목록 필터로 처리
- 최근 검색어 저장·삭제 지원

### 정책정보

- 청년 취업지원
- 직업훈련·교육
- 창업 지원
- 잡아바 일자리 플랫폼

## 관리자 화면

### 통계

- 전체 적재 공고 수
- 현재 사용자 화면 노출 공고 수(마감 제외)
- 마감 공고 수
- 출처/지역/경력/학력/고용형태 통계

### 채용공고 CRUD

- 공고 목록 검색·페이징
- 신규 등록
- 수정
- 삭제
- 등록/수정 시 관리자는 지역과 주소만 입력한다.
- 서버가 주소→좌표 지오코딩을 수행하고 실패 시 지역 중심점으로 보정한다.
