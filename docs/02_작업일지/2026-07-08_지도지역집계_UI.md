# 지도 지역 집계 UI 개선 (2026-07-08)

- 지도에 개별 채용공고 핀 대신 현재 지도 범위·검색·필터 결과를 첫 근무지역 기준으로 묶어 `공고 수 + 지역명` 배지로 표시했다.
- 여러 공고가 있는 지역 배지는 클릭 시 해당 위치로 확대하고, 단일 공고 배지는 기존처럼 공고를 선택하도록 했다. 카드 선택 시 소속 지역 배지도 선택 상태로 강조된다.
- PC에서 왼쪽 공고 카드에 마우스를 올리면 지도에 해당 좌표를 반투명 원으로 표시한다. 정밀 좌표는 700m, 지역 근사 좌표는 2.2km 반경으로 정확도 차이를 반영했다.
- 지도 레벨 3 이하로 아주 가깝게 확대했을 때만 지역 집계 배지를 공공/민간 개별 공고 핀으로 전환하고, 지도 우측에 `+ / −` 확대·축소 버튼을 추가했다.
- 개별 핀 클릭 시 왼쪽 목록을 해당 공고 1건으로 좁히고, 휠이나 버튼으로 줌 레벨이 바뀌면 현재 지도 범위의 전체 목록으로 자동 복원하도록 상태 흐름을 보완했다.
- 개별 핀이 표시되는 지도 레벨 3 이하에서는 카드 hover 반경 원을 생략해, 확대된 지도에서 원이 화면을 과도하게 덮는 문제를 수정했다.
- 운영 코드는 지역 집계 UI가 유지되고 있었지만 전환 기준이 너무 이른 레벨 5였음을 확인했다. 중간 줌에서도 `개수 + 지역명`을 유지하도록 레벨 3으로 조정하고 CSS/JS에 버전을 붙여 이전 캐시가 롤백처럼 보이는 문제를 방지했다.
- 지역 배지 hover 시 배지에 포함된 모든 공고 좌표를 감싸는 반투명 범위를 표시한다. 행정경계 데이터 대신 실제 포함 공고의 중심~최장 거리에 여유 반경을 더해 계산한다.
- 지역 배지 클릭 시 자동 줌 대신 왼쪽 목록을 해당 지역 공고로 한정하고, 이후 줌 레벨이 변경되면 지역 선택을 해제해 현재 지도 영역의 전체 공고 목록을 복원한다.
- 복수 근무지의 첫 문자열 때문에 성남 좌표에 `강남구 외 14`가 표시되던 문제를 수정했다. 통계청 원천 경기도 시·군·구 GeoJSON에 좌표를 point-in-polygon으로 대입해 실제 위치 명칭으로 집계한다(확인한 문제 공고 좌표는 `성남시 수정구`).
- 지역 배지 hover는 공고 좌표를 감싼 원 대신 해당 시·군·구의 실제 행정경계 MultiPolygon을 반투명하게 표시한다.
- 와이즈만 수학/과학 공고가 사용자 브라우저에서 같은 페이지로 열리는 현상에 대응해 잡코리아 링크의 추적 쿼리를 제거하고 공고번호 전용 URL로 정규화했다. 링크 DOM에도 공고 ID를 명시해 서로 다른 공고임을 보장한다.
- 현재 위치가 부천 등 다른 도시로 잡히는 문제를 재점검했다. 단일 브라우저 위치값을 즉시 적용하던 방식을 12초간 다중 측정 후 최적 정확도 선택으로 바꾸고, 오차 3km 초과 위치는 지도 이동을 취소하도록 보완했다.
- 가까운 줌에서 동일 공고가 여러 번 보이는 원인을 운영 API에서 재현했다(527행 중 고유 ID 377개). 등록일 단일 정렬로 페이징 순서가 흔들리던 문제를 `등록일 DESC, ID DESC` 안정 정렬로 수정하고, 프런트에도 ID 중복 제거를 추가했다.
- 잡코리아 공고 `49360260`(에듀플렉스 고잔점)은 공식 지점 주소와 좌표(37.308362, 126.832064)를 원본 공고 ID 기반 예외 데이터로 반영해 `exact` 위치로 표시되도록 수정했다.
- 전체 600건 원본을 재점검해 동일 회사·제목·근무지역으로 재등록된 잡코리아 3건은 최신 공고만 유지했다. 사업자번호가 일치하는 제공 기업 좌표도 우선 적용하고, 나머지는 상세주소 지오코딩 또는 지역 근사임을 구분했다.
- 백엔드 재기동 검증에서 시드 적재기가 실행되지 않아 API가 0건인 문제를 발견했다. 적재 대상을 `local`, `prod` 프로필로 명시해 재기동 때 정제된 전체 데이터로 DB가 확실히 교체되도록 수정했다.
