# 프로젝트 구현 현황 정리 (2026-07-05)

## 목표

- 고객 요청사항을 근거로 분석 → 설계 → 개발 → QA → 서버 배포까지 완주한다.
- 지도 기반 채용공고 탐색 서비스를 팝업 위젯 형태로 제공한다.
- 산출물은 `docs/` 하위에 요구사항, 설계, QA, 배포 문서로 정리한다.

## 구현 완료 항목

- 프런트 메인 화면: 검색, 필터, 좌측 목록, 우측 지도, 모바일 반응형 화면을 구현했다.
- 카카오맵 연동: 지도 표시, 마커/지역 배지, 카드와 지도 상호작용, 줌/드래그 갱신을 구현했다.
- 데이터 파이프라인: 원본 CSV를 표준 파싱 기준으로 정제하고 중복·결측·형식 오류를 검증해 `data/processed/jobs.json`을 생성했다.
- 백엔드 API: 채용공고 조회, 근처 공고, 상세, 코드, 관리자 통계·CRUD API를 Spring Boot로 구현했다.
- 관리자 인증: `/api/admin/**`에 HTTP Basic 인증을 적용하고 운영 자격증명을 환경변수로 분리했다.
- 배포 구성: EC2 단일 인스턴스에서 nginx, Spring Boot, PostgreSQL/PostGIS를 Docker Compose로 운영하도록 구성했다.
- 운영 배포: `https://jobmapkorea.com`에 사용자 화면, 관리자 화면, 팝업 위젯을 배포했다.

## 데이터 처리 결과

- 표준 파싱 원본: 고용24 100건, 잡코리아 500건.
- 운영 적재: 전체 460건.
- 사용자 노출: 현재일 기준 마감 공고 제외 336건.
- 마감 공고: 124건.
- 좌표 정밀도: `exact` 25건, `company_address` 97건, `region_approx` 338건.

## 좌표 기준

- 고용24는 `BASIC_ADDR + DETAIL_ADDR`를 근무지 주소로 사용한다.
- 잡코리아는 `BIZ_NO`와 기업주소 자료의 `BIZRNO`를 조인해 주소를 보완하고, 실패 시 `AREA_INFO` 지역 중심점으로 보정한다.
- 관리자 등록/수정은 위도·경도 직접 입력 없이 지역과 주소를 입력하면 서버가 좌표를 자동 생성한다.
