## 확정된 기술 스택 / 아키텍처

- **백엔드**: Spring Boot + **Gradle** (기존 프로젝트가 Gradle이라 통일)
- **DB**: PostgreSQL + PostGIS (지도 bbox 조회, 반경 검색, 거리순 정렬에 필요)
- **프런트**: 순수 HTML/JS/jQuery + 카카오맵 SDK (팝업 이식 방식이라 무거운 빌드 도구 없이)
- **배포**: ~~Cloudflare Pages + AWS EC2 분리~~ → **AWS EC2 단일 인스턴스로 통합 결정**
  - EC2 안에 Docker Compose로 nginx(정적 프런트 서빙 + `/api` 리버스 프록시) + backend + db(PostGIS) 세 컨테이너
  - CORS 불필요 (같은 오리진)
  - 도메인은 아직 미구매 — Elastic IP만 먼저 발급받아 EC2에 고정해두는 걸 권장한 상태
- **개발 도구**: IntelliJ + Claude Code로 **백엔드/프런트/인프라 전부 통일** (Cursor, VS Code는 안 씀. Gradle 확장자 연결 체크 완료)
- **Antigravity**: 메인 개발엔 안 쓰고, 나중에 QA/브라우저 검증 단계에서만 보조로 사용 예정
- **디자인**: Figma 에이전트/MCP 검토했으나, 요금(에이전트는 Professional 유료 필요) 및 프로젝트 규모(화면 4~5개) 감안해 **Figma 없이 Claude Code에게 직접 화면 구현을 요청하는 방식으로 결정**. Indeed·잡코리아·원티드 UI 패턴을 참고 레퍼런스로 삼기로 함.
