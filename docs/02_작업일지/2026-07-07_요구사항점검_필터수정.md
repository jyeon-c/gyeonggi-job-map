# 요구사항 점검 및 필터 수정 (2026-07-07)

## 점검 결과

- 고객 요청 10건의 구현을 문서·프런트·백엔드·배포 구성과 대조했다. 1, 2, 4~10번은 구현되어 있으며, 3번의 PC IP 기반 기본 위치는 기존 문서대로 미구현(경기 중심 기본값)이다.
- 설계 문서가 요구한 PostGIS 공간 쿼리(`ST_MakeEnvelope`, `ST_DWithin`)는 아직 위·경도 범위 조회와 Java 거리 계산으로 대체되어 있다. 현재 527건에서는 기능상 동작하지만 데이터 증가 시 전환이 필요하다.
- 원본 데이터와 비밀 파일의 gitignore, Docker Compose의 nginx/backend/PostGIS 구성, 운영 환경변수 주입을 확인했다.

## 수정 및 검증

- 프런트의 마감 판정 기준일이 `2026-07-05`로 고정되어 시간이 지나면 마감 공고가 노출되는 문제를 실행 당일의 로컬 날짜 기준으로 수정했다.
- 학력·고용형태·직종·희망임금 복합 필터와 희망임금 경계값 API 회귀 테스트를 추가했다.
- 운영 확인 중 기존 DB가 최초 시드 이후 갱신되지 않아 직종·최소연봉 컬럼이 비어 있는 원인을 발견했다. 기동 시 `jobs.json`을 한 트랜잭션으로 교체하도록 로더를 수정해 데이터 스키마·분류 변경도 운영에 반영되게 했다.
- IntelliJ JBR 21로 Gradle 전체 테스트를 실행해 통과를 확인했다.
