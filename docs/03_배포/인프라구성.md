# 인프라 구성 (EC2 통합)

## 구조

```
EC2 (단일 인스턴스)
├── nginx 컨테이너   → 정적 프런트 서빙 + /api 리버스 프록시
├── backend 컨테이너 → Spring Boot
└── db 컨테이너      → PostgreSQL + PostGIS
```

## docker-compose.yml 구성 요소 (구현 완료)

- [x] nginx: `infra/nginx/Dockerfile` 로 `frontend/` 를 이미지에 포함, `/api/` → `backend:8080` 프록시(`infra/nginx/default.conf`). 기동 시 `30-render-config.sh` 가 환경변수로 `js/config.js`(카카오 키·apiBase) 생성.
- [x] backend: `backend/Dockerfile`(멀티스테이지, 빌드 컨텍스트=루트). `prod` 프로필 + `DB_URL/DB_USERNAME/DB_PASSWORD` 환경변수 주입(하드코딩 없음). 이미지에 `jobs.json` 포함 → 기동 시 `JobDataLoader`가 PostgreSQL 데이터를 검증된 시드로 교체.
- [x] db: `postgis/postgis:16-3.4`, `pgdata` 볼륨으로 영속화, healthcheck 로 backend 기동 순서 보장.
- [x] 세 컨테이너는 compose 기본 네트워크로 연결(`db`/`backend` 서비스명으로 통신).

### 실행
```bash
cp .env.example .env      # KAKAO_JS_KEY, DB_PASSWORD 등 채우기
docker compose up -d --build
# http://<EC2-IP>/ 접속
```

> 참고: prod 는 현재 `ddl-auto: update` 로 스키마 자동 생성(실습 단계). 좌표는 lat/lng 컬럼으로
> 운용하며 PostGIS 도입(geometry+GiST)은 `JobPostingRepository`/`Service` 주석의 전환 지점 참고.
> ⚠️ 카카오 콘솔 [플랫폼 > Web]에 EC2 도메인/IP(예: `http://<EC2-IP>`)를 추가 등록해야 지도가 뜬다.

## HTTPS

- 운영 도메인 `jobmapkorea.com`은 Cloudflare 프록시를 통해 HTTPS 제공.
- 현재 Flexible 구성은 추후 origin 인증서 + Full(Strict)로 강화 권장.

## PC IP 기본 위치

- Cloudflare Dashboard → Rules → Transform Rules → Managed Transforms에서 `Add visitor location headers` 활성화.
- 원 서버로 전달되는 `CF-IPLatitude/CF-IPLongitude`를 `/api/location`이 검증해 사용한다.
- 헤더 미제공이나 직접 IP 접속에서는 브라우저 Geolocation으로 폴백한다.

## 환경변수 관리

- `.env` 파일은 `.gitignore`에 반드시 포함
- `.env.example`만 커밋해서 어떤 값이 필요한지 공유

## 배포 절차 (수동, 추후 CI/CD로 대체 가능)

1. `git pull`
2. `docker compose build`
3. `docker compose up -d`
4. 로그 확인: `docker compose logs -f backend`

## 향후 개선 아이디어 (선택)
