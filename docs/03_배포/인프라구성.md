# 인프라 구성 (EC2 통합)

## 구조

```
EC2 (단일 인스턴스)
├── nginx 컨테이너   → 정적 프런트 서빙 + /api 리버스 프록시
├── backend 컨테이너 → Spring Boot
└── db 컨테이너      → PostgreSQL + PostGIS
```

## docker-compose.yml 구성 요소 (체크리스트)

- [ ] nginx: frontend 빌드 결과물 볼륨 마운트, `/api/` → `backend:8080` 프록시
- [ ] backend: `application.yml`은 환경변수로 DB 접속정보 주입 (하드코딩 금지)
- [ ] db: `postgis/postgis` 이미지, 볼륨으로 데이터 영속화
- [ ] 세 컨테이너 모두 같은 네트워크에 연결

## HTTPS

- Let's Encrypt(certbot)로 인증서 발급 → nginx에 적용
- 인증서 자동 갱신 cron 등록

## 환경변수 관리

- `.env` 파일은 `.gitignore`에 반드시 포함
- `.env.example`만 커밋해서 어떤 값이 필요한지 공유

## 배포 절차 (수동, 추후 CI/CD로 대체 가능)

1. `git pull`
2. `docker compose build`
3. `docker compose up -d`
4. 로그 확인: `docker compose logs -f backend`

## 향후 개선 아이디어 (선택)

- GitHub Actions로 push 시 EC2 자동 배포(SSH 접속 후 위 절차 실행)
