# 관리자 검수 체크리스트

이 문서는 관리자가 운영 서비스와 산출물을 순서대로 확인하기 위한 체크리스트다.

운영 데이터 검수 기준일: 2026-07-09

## 1. 산출물 구조 확인

| 확인 항목 | 기준 | 상태 |
|---|---|---|
| 요구사항 문서 | 고객 요청 10건과 구현 결과가 정리되어 있음 | 완료 |
| 설계 문서 | 시스템, 화면, API, ERD가 분리되어 있음 | 완료 |
| 소스 산출물 | `frontend/`, `backend/`, `scripts/`, `data/processed/`가 존재함 | 완료 |
| QA 문서 | 요구사항 추적표와 테스트 결과가 존재함 | 완료 |
| 배포 문서 | 인프라, EC2 배포, 운영 배포 결과, DB 백업 문서가 존재함 | 완료 |

## 6. API 검수

| API | 확인 목적 |
|---|---|
| `GET /api/jobs` | 사용자 채용공고 목록 조회 |
| `GET /api/jobs/nearby` | 현재 위치 기준 주변 공고 조회 |
| `GET /api/jobs/{id}` | 공고 상세 조회 |
| `GET /api/codes/{group}` | 필터 코드 조회 |
| `GET /api/location` | PC IP 위치 또는 브라우저 위치 폴백 확인 |
| `GET /api/admin/stats` | 관리자 통계 조회 |
| `GET /api/admin/jobs` | 관리자 공고 목록 조회 |
| `POST /api/admin/jobs` | 관리자 공고 등록 |
| `PUT /api/admin/jobs/{id}` | 관리자 공고 수정 |
| `DELETE /api/admin/jobs/{id}` | 관리자 공고 삭제 |

## 8. 제출 전 최종 확인

- `docs/README.md`를 먼저 열어 산출물 순서를 확인한다.
- 요구사항 분석 문서에서 고객 요청 10건이 모두 구현 완료인지 확인한다.
- QA 문서에서 요구사항별 검증 결과를 확인한다.
- 운영 배포 결과 문서에서 실제 서비스 URL과 관리자 화면 배포 상태를 확인한다.
- 관리자 화면에서 통계와 CRUD가 실제로 동작하는지 확인한다.
