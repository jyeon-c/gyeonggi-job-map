## 2. 산출물 확인 순서

### 1단계. 요구사항 확인

| 문서 | 목적 |
|---|---|
| `00_요구사항/00_실습과제_명세서.md` | 실습 과제의 목표와 제출 기준 확인 |
| `00_요구사항/고객_요청사항.xlsx` | 고객 원본 요청사항 확인 |
| `00_요구사항/고객요청사항_분석.md` | 고객 요청 10건별 구현 결과 확인 |

확인할 내용:

- 반응형 팝업, 지도 표시, 현재 위치, 주소 검색, 채용 목록, 검색·필터, 정책 링크, 데이터 연계, 보안 요구사항이 모두 구현 완료로 정리되어 있는지 확인한다.
- 관리자 통계와 채용공고 CRUD는 추가 요청으로 구현된 기능임을 확인한다.

### 2단계. 설계 산출물 확인

| 문서 | 목적 |
|---|---|
| `01_설계/시스템구성.md` | 전체 서비스 구조, 데이터 흐름, 관리자 데이터 흐름 확인 |
| `01_설계/화면설계.md` | 사용자 화면과 관리자 화면 기능 확인 |
| `01_설계/API명세.md` | 사용자·관리자 API 명세 확인 |
| `01_설계/ERD.md` | DB 테이블 구조와 좌표 처리 기준 확인 |

확인할 내용:

- 사용자 화면은 검색, 필터, 지도, 목록, 정책정보, 위치 기능을 포함한다.
- 관리자 화면은 통계와 채용공고 등록·수정·삭제를 포함한다.
- 관리자 등록/수정은 위도·경도 직접 입력이 아니라 지역과 주소 입력 기반으로 좌표를 생성한다.
- 데이터 좌표는 주소 지오코딩 우선, 실패 시 지역 중심점 보정 원칙을 따른다.

### 3단계. 소스 산출물 확인

| 경로 | 역할 |
|---|---|
| `frontend/` | 사용자 화면, 관리자 화면, 팝업 위젯 정적 리소스 |
| `backend/` | Spring Boot API, 관리자 인증, DB 적재, 좌표 생성 |
| `scripts/` | CSV 정제, jobs.json 생성, 운영 DB 백업 스크립트 |
| `data/processed/jobs.json` | 정제 후 운영 시드 데이터 |
| `data/processed/data-quality-report.json` | 데이터 정제 품질 리포트 |
| `docker-compose.yml` | 운영 컨테이너 통합 구성 |
| `infra/nginx/` | 정적 파일 서빙과 `/api` 리버스 프록시 |

확인할 내용:

- 원본 개인정보 파일과 비밀값 파일은 커밋 대상이 아니다.
- 운영 시드 데이터는 중복·결측·형식 검증을 통과한 결과만 사용한다.
- 카카오 REST 키와 DB/관리자 비밀번호는 환경변수로 주입한다.

### 4단계. QA 산출물 확인

| 문서 | 목적 |
|---|---|
| `02_QA/요구사항_추적표_및_QA결과.md` | 요구사항별 구현 추적과 테스트 결과 확인 |
| `02_작업일지/2026-07-05_프로젝트현황.md` | 전체 구현 흐름과 최종 상태 확인 |
| `02_작업일지/2026-07-07_요구사항점검_필터수정.md` | 요구사항 점검과 필터 수정 이력 확인 |
| `02_작업일지/2026-07-08_지도지역집계_UI.md` | 지도 지역 집계 UI 구현 이력 확인 |
| `02_작업일지/2026-07-09_근무지기준_좌표정제.md` | 데이터 좌표 정제 이력 확인 |

확인할 내용:

- 사용자 노출 건수는 서비스 실행일 기준 마감 공고를 제외한다.
- 관리자 통계는 전체 적재 건수, 마감 제외 건수, 마감 건수를 확인할 수 있다.
- 좌표 결측, 중복, 형식 오류가 없는 정제 데이터를 사용한다.

### 5단계. 배포 산출물 확인

| 문서 | 목적 |
|---|---|
| `03_배포/인프라구성.md` | 운영 인프라 구조 확인 |
| `03_배포/EC2_배포_가이드.md` | EC2 배포 절차와 운영 명령 확인 |
| `03_배포/2026-07-08_운영배포결과.md` | 운영 반영 결과 확인 |
| `03_배포/DB_백업_운영가이드.md` | DB 백업과 복구 절차 확인 |

확인할 내용:

- 운영은 EC2 단일 인스턴스에서 nginx, backend, PostgreSQL/PostGIS 컨테이너로 구성된다.
- Cloudflare를 통해 HTTPS가 제공된다.
- 운영 DB 백업은 `~/jobmap-backups`에 생성된다.

## 3. 구현 기능 요약

### 사용자 기능

- 팝업 위젯 삽입
- PC/모바일 반응형 지도 화면
- 분양처별 브랜딩
- 공공/민간 공고 구분
- 지도 줌·드래그에 따른 목록 자동 갱신
- 지역별 공고 수와 지역명 배지 표시
- 가까운 줌에서 개별 공고 표시
- 지역 hover 시 행정구역 범위 표시
- 공고 마커 클릭 시 해당 공고 1건 표시
- 현재 위치 이동과 위치 정확도 안내
- 주소·장소 검색과 최근 검색어
- 직종, 경력, 학력, 고용형태, 희망임금 필터
- 마감 공고 제외 노출
- 정책정보 링크 제공

### 관리자 기능

- 관리자 로그인
- 전체 적재 공고 수 확인
- 사용자 화면 노출 공고 수 확인
- 마감 공고 수 확인
- 출처, 지역, 경력, 학력, 고용형태 통계 확인
- 채용공고 검색·페이징
- 채용공고 등록
- 채용공고 수정
- 채용공고 삭제
- 등록/수정 시 주소 기반 좌표 자동 생성

### 데이터 기능

- 고용24와 잡코리아 CSV 표준 파싱
- 공통코드·직종분류 매핑
- 중복 제거
- 필수값 결측 검증
- 날짜·URL·좌표 형식 검증
- 고용24 상세주소 기준 지오코딩
- 잡코리아 기업주소 조인 기반 지오코딩
- 실패 시 지역 중심점 보정
