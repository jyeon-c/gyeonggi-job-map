# 화면설계 — Figma 에이전트 → Figma MCP → Claude Code 워크플로우

## 0. 준비

- Figma Design 파일 생성: `경기도 일자리맵 - Design`
- Figma 설정 → Security → Personal Access Token 발급 (MCP 연결용)
- Claude Code에 Figma MCP 서버 등록 (`claude mcp add figma ...`)

## 1. Figma 에이전트에게 화면별로 순서대로 요청 (한 번에 통으로 X)

화면을 나눠서 하나씩 만들고, 각 결과를 검토한 뒤 다음으로 넘어간다.

### 1) 전체 레이아웃 탐색 (스타일 방향 결정용)
```
지도 기반 채용공고 탐색 팝업 위젯의 스타일을 3가지 방향으로 제안해줘.
하나는 깔끔한 공공서비스 느낌(파란색 계열, 정보 밀도 높음),
하나는 스타트업 채용 서비스 느낌(모던, 여백 많음),
하나는 지도앱 느낌(초록/주황 포인트, 마커 중심).
데스크톱 1280px 기준으로 3개를 나란히 보여줘.
```

### 2) 지도 메인 화면
```
지도 기반 채용공고 탐색 화면을 만들어줘.
- 상단: 검색창 + 필터 버튼(경력/학력/고용형태/공공·민간)
- 좌측(데스크톱) 또는 하단(모바일): 채용공고 리스트, 카드형
- 카드에는 회사명, 공고 제목, 거리(km), 공공/민간 뱃지
- 지도 영역에는 마커 표시(공공=파란색, 민간=주황색)
- 우측 상단에 "내 위치로" 버튼
데스크톱(1280px)과 모바일(390px) 두 가지 크기로 만들어줘.
```

### 3) 상세 패널
```
지도 화면에서 마커 또는 리스트 카드를 클릭했을 때 나오는 상세 정보 패널을 만들어줘.
데스크톱은 우측 슬라이드 패널, 모바일은 하단 바텀시트 형태로.
포함 정보: 공고 제목, 회사명, 주소, 경력/학력/고용형태 뱃지, 지원 버튼(외부 링크 아이콘 포함).
```

### 4) 필터 UI
```
경력/학력/고용형태/출처(공공·민간)를 선택하는 필터 UI를 만들어줘.
데스크톱은 드롭다운 다중선택, 모바일은 하단에서 올라오는 필터 시트 형태로.
선택된 필터는 상단에 칩(chip) 형태로 표시.
```

### 5) 반복/수정 요청 예시 (에이전트의 강점 영역)
```
리스트 카드 컴포넌트를 전체 화면에 일괄 적용하고,
패딩을 16px로 통일하고, 실제 채용공고처럼 보이는 더미 텍스트로 채워줘.
```

## 2. Figma MCP로 Claude Code에 연결

1. Figma에서 완성된 프레임 우클릭 → "Copy link to selection"
2. Claude Code 터미널에 링크 붙여넣고 아래처럼 요청

```
이 Figma 프레임(링크)을 기준으로 반응형 HTML/CSS/JS로 구현해줘.
- 빌드 도구 없이 순수 HTML/JS/JQuery
- 카카오맵 SDK는 <script> 태그로 로드
- CSS는 별도 파일(css/responsive.css)로 분리
- 지도 마커 클릭 시 상세 패널 열리는 인터랙션까지 JS로 구현
- 컴포넌트 구조는 Figma의 레이어 이름을 최대한 그대로 반영해줘 (나중에 디자인 변경 시 추적 쉽게)
```

### 화면 단위로 나눠서 요청 (권장)
한 번에 "전체 페이지 만들어줘"라고 하면 품질이 떨어진다. 아래처럼 나눈다.

```
1) 지도 메인 화면 마크업/CSS만 먼저 구현해줘
2) 리스트 카드 컴포넌트 구현해줘 (지도 화면에 삽입)
3) 상세 패널(데스크톱 슬라이드/모바일 바텀시트) 구현해줘
4) 필터 UI 구현하고 지도 화면에 연결해줘
```

## 3. 주의사항

- Code Connect 없이는 spacing/색상 정확도가 완벽하지 않을 수 있음 → 실제 값과 대조하며 수동 보정 필요
- Figma 파일 링크, 디자인 결정 사항은 이 문서 하단 "결정 로그"에 계속 기록
- 완성된 화면 스크린샷은 `docs/01_설계/screenshots/`에 저장 (선택)
